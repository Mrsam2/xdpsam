# Onboarding

> Goal: every new team member is productive, connected, and confident by the end of week one, and fully ramped by day 90.

## Before day 1 (hiring manager checklist)

- [ ] Signed agreement + IP assignment + NDA on file.
- [ ] Accounts created (email, GitLab, chat, password manager) with least-privilege access.
- [ ] Hardware/access ready.
- [ ] Buddy/mentor assigned.
- [ ] First-week plan and 30/60/90 goals drafted.

## Day 1

- Welcome + company story, vision, mission, values (`company/vision-mission-values.md`).
- Tour of this repository as the operating manual.
- Access verification and security basics (`security/security-policy.md`).
- Meet the team; introduce the buddy.

## Week 1

- Read core docs: values, code of conduct, engineering standards (if technical), security policy.
- Complete first small, real task (a "first MR" or first client-facing deliverable).
- 1:1 with manager to clarify expectations.

## 30 / 60 / 90 day plan

| Milestone | Expectation |
|---|---|
| **Day 30** | Understands role, tooling, and team; delivering small tasks independently. |
| **Day 60** | Owning meaningful work; contributing in their domain; knows the customer. |
| **Day 90** | Fully ramped; delivering against role outcomes; giving and receiving feedback. |

## Offboarding (for completeness)

- Revoke all access immediately on departure.
- Recover hardware and assets.
- Knowledge transfer / documentation handover.
- Final pay, equity treatment per vesting, and exit interview.
