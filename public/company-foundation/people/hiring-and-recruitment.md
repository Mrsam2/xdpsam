# Hiring & Recruitment Playbook

> How XDP technologies finds, evaluates, and hires people. Designed to be fair, fast, and bias-aware.

## Hiring philosophy

- **Hire for slope, not just intercept.** Trajectory and learning rate beat current snapshot.
- **Values fit is non-negotiable.** Skills can be taught; values are screened hard.
- **Hire when the pain is real.** Each hire must map to a concrete need and budget. No vanity headcount.
- **Default to small, senior, generalist early.** Specialize as the company scales.

## When to open a role

1. A founder/lead identifies a recurring bottleneck that a role would resolve.
2. Budget and runway impact confirmed (see `finance/budgeting-and-runway.md`).
3. Founders approve (Tier 1 decision).
4. Write the role spec before sourcing.

## Role spec template

```
# Role: <title>
- Mission: <one sentence on why the role exists>
- Outcomes (first 12 months): <3-5 measurable outcomes>
- Must-have competencies:
- Nice-to-have:
- Level / band:
- Compensation range:
- Hiring manager + interview panel:
```

## Standardized hiring process

| Stage | Goal | Owner |
|---|---|---|
| 1. Application screen | Filter for must-haves | Hiring manager |
| 2. Intro call (30 min) | Mutual fit, motivation, comp alignment | Hiring manager |
| 3. Skills assessment | Verify real ability (work sample / paid trial task) | Panel |
| 4. Team & values interview | Collaboration + values fit | 2 interviewers |
| 5. Founder conversation | Vision alignment, final read | Co-founder |
| 6. References + offer | Verify + close | Hiring manager |

## Evaluation principles

- Use a **structured scorecard** per competency; score independently before debriefing.
- Prefer **work samples** over abstract puzzles.
- Decisions are **evidence-based**: "strong yes / yes / no / strong no" with written rationale.
- A single "strong no" on values from any interviewer blocks the hire.

## Inclusive hiring

- Write inclusive job descriptions; avoid gendered or exclusionary language.
- Source from diverse channels.
- Standardize questions and scorecards to reduce bias.

## Contractors vs. employees

- Use contractors for bounded, specialized, or surge work.
- Convert to employee when the need is ongoing and core.
- Always sign IP assignment + NDA before any work starts.
